## Finding 1: Whale Concentration Is Systemic (92-98% Across All Epochs)

Whale concentration (top 10% of donors controlling majority of funding) is a structural feature of Octant, not a temporary anomaly:

| Epoch | Donations | Unique Donors | Total ETH | Whale Concentration | Max Single Donation |
|-------|-----------|---------------|-----------|--------------------|--------------------|
| 4 | 1,602 | 279 | 4.79 ETH | **92.8%** | 0.25 ETH |
| 5 | 1,902 | 422 | 17.63 ETH | **97.9%** | 2.05 ETH |
| 6 | 1,978 | 455 | 14.31 ETH | **97.2%** | 1.84 ETH |

**Insight:** Despite growing donor counts (279 to 455, +63% over 3 epochs), whale concentration remains above 92% in every epoch. The median donation is ~0.00015 ETH (~$0.37) while the max is 1,000-14,000x larger. Quadratic funding matching is driven by a handful of large allocators, not broad community preference.

**What this means for evaluators:** Simple "number of donors" is a misleading quality signal. A project with 400 donors may have 97% of its funding from 40 wallets.

## Finding 7: Signal Reliability Varies 10x Between Sources

Tessera's Signal Quality Framework classifies every data point by trustworthiness:

| Tier | Examples | Can Be Faked? | Cost to Fake |
|------|----------|---------------|--------------|
| **HIGH** | On-chain balance (RPC), Octant allocations (protocol), Shannon entropy (computed), Jaccard similarity (computed) | No | Impossible (immutable) or requires changing real donations |
| **MEDIUM** | OSO contributor counts (~24h lag), GitHub commit frequency, Discourse engagement | Partially | Medium (bot commits, fake accounts) |
| **LOW** | GitHub stars (star farms), self-reported proposal claims | Yes | Low ($50 for 1000 stars) |

**Signal corroboration reveals discrepancies:** When OSO reports 17 contributors but GitHub shows 30, that's a freshness gap (OSO has ~24h indexing lag). When a proposal claims "$10M distributed" but on-chain data shows $32M, that's an understated claim (Protocol Guild scan-proposal finding).

**Data freshness matters:** Blockchain data is real-time (latest block). GitHub `pushed_at` is seconds-old. OSO metrics have ~24h lag. A stale signal can contradict a fresh one without either being wrong.

## Methodology

All findings are generated from live Octant API data using deterministic algorithms in Tessera's 11-step pipeline:

| Algorithm | Purpose | Mathematical Basis |
|-----------|---------|-------------------|
| Whale concentration | Top 10% donor share | Percentile threshold on sorted donation distribution |
| Shannon entropy | Donor diversity | H = -Sum(p_i * ln(p_i)) / ln(n), normalized 0-1 |
| Jaccard similarity | Coordination detection | |A intersect B| / |A union B| on donor address sets |
| Union-find clustering | Donor grouping | Graph-based clustering with Jaccard >0.7 threshold |
| K-means | Project clustering | Lloyd's algorithm on (allocated, matched) feature space |
| Composite scoring | Basic ranking | 40% normalized allocated + 60% normalized matched, scaled 0-100 |
| Multi-layer scoring | Trust-adjusted ranking | 25% funding + 25% efficiency + 30% diversity + 20% consistency |
| Trust-Weighted QF | Novel mechanism | score = qf_score * (0.5 + 0.5 * diversity) |
| Gini coefficient | Inequality measurement | Sum|x_i - x_j| / (2n * total) |
| Signal corroboration | Cross-verification | 7 checks across independent sources with severity-weighted scoring |
| Temporal anomalies | Pattern changes | Cross-epoch delta analysis with union-find coordination shift detection |

**Data sources (9):** Octant REST API, Gitcoin GraphQL, OSO GraphQL, GitHub REST, 9 EVM chain RPCs, Block Explorers, Octant Discourse, Optimism RetroPGF, Optimism Gov Forum.

**AI evaluation:** Claude Opus 4.6 via Claude CLI, fed with ALL quantitative evidence from steps 1-8 before generating narrative assessment. The AI does not generate numbers — it interprets pre-computed metrics.
